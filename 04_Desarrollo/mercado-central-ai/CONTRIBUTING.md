# Contributing

## Flujo de trabajo

1.  Crear o actualizar el requisito funcional.
2.  Actualizar la documentación (DOC/SDS).
3.  Implementar el código.
4.  Crear o actualizar pruebas.
5.  Actualizar MTR y LOG.
6.  Ejecutar las pruebas.
7.  Realizar commit descriptivo.
8.  Crear tag cuando corresponda.

## Convenciones

-   Código en inglés.
-   Documentación en español.
-   Una responsabilidad por módulo.
-   Configuración centralizada en `settings.py`.

## Commits

Formato sugerido:

    Sprint X: descripción breve
